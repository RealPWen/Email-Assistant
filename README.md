# Email Dashboard - 邮件同步与审计系统

本项目是一个基于 Python 的高性能邮件抓取与本地持久化系统，旨在为用户提供快速、稳定且智能的邮件存档方案，支持后续 Dashboard 展示与 AI 深度分析。

## 🌟 核心功能特性

### 1. ⚡ 高性能批量同步 (Batch Sync)
针对传统 IMAP 同步缓慢的问题，本项目实现了多阶段批量处理逻辑：
- **快速扫描 (Fast Scan)**：以 50 封为一组批量拉取邮件头（Header）和状态（Flags），实时比对数据库记录，仅对缺失部分发起下载请求。
- **批量抓取 (Batch Fetch)**：以 20 封为一组批量拉取 RFC822 完整内容，极大地降低了网络往返延迟（RTT），同步效率比普通方案快数倍。

### 2. 📅 时间数据标准化 (Date Normalization)
自动将邮件头中多变的 RFC 2822 日期格式（如 `Sat, 11 Apr 2026...`）转换为标准的 **ISO 8601 时间戳**。
- **意义**：支持在 SQLite 中直接进行按日期范围查询、按月/周进行趋势统计，以及前端图表的时间轴渲染。

### 3. 📩 状态自动化同步 (Status Sync)
系统能够与邮件服务器实时同步状态变化。
- **实时更新**：每次运行脚本都会在扫描范围内更新邮件的 `is_read`（已读/未读）标志，确保本地 Dashboard 与 163 等远程收件箱状态完全一致。

### 4. 📎 附件元数据管理 (Metadata Extraction)
在不下载巨大附件的情况下，深度解析邮件结构：
- **元数据存储**：提取附件的文件名、文件大小及 MIME 类型，并以 JSON 存储在 `attachments_metadata` 字段中。
- **Dashboard 集成**：支持在列表页显示附件标识，并在详情页显示附件清单，兼顾性能与功能完整性。

### 5. 🛡️ 稳健的架构设计
- **去重机制**：使用全局唯一的 `Message-ID` 作为主键，从根本上杜绝重复邮件。
- **零成本迁移**：数据库内置版本检测与自动升级逻辑，新增功能时无需手动操作数据库即可完成架构更新。

## 🛠️ 实现逻辑

### 数据流向
1. **连接与鉴权**：通过 `IMAP4_SSL` 连接服务器，并发送 `XATOM ID` 识别包。
2. **增量扫描**：从最新邮件开始反向扫描 `max_scan` 范围内的邮件。
3. **内容解析**：使用 `email.message_from_bytes` 递归解析 MIME 树，提取纯文本正文。
4. **持久化**：将标准化后的数据写入 `data/emails.db` (SQLite)。

### 关键路径处理
- **无乱码解码**：基于 `email.header.decode_header` 处理多种字符集编码（GBK, UTF-8 等）。
- **递归遍历**：利用 `msg.walk()` 确保在复杂的多级 multipart 邮件中准确提取目标内容。

## 🎯 它可以完成的任务
- **邮件本地归档**：建立永久的本地邮件库，防止邮件被误删或服务器策略性清除。
- **趋势分析**：通过 SQL 统计发件人频率、处理时段分布等。
- **审计与搜索**：利用 SQLite 的高效索引，秒级检索万封邮件中的关键词。

## 🏗️ 快速开始

### 1. 配置环境
在根目录创建 `.env` 文件并填写配置：
```ini
EMAIL_USER=your_email@163.com
EMAIL_AUTH_CODE=your_authorization_code
```

### 2. 运行同步
```bash
python3 tools/fetch_emails.py
```

### 3. 数据库结构
- `id`: 自增主键
- `message_id`: 唯一标识
- `subject`: 邮件主题
- `sender`: 发件人信息
- `normalized_date`: ISO 标准时间
- `body`: 邮件正文
- `is_read`: 已读/未读 (1/0)
- `attachments_metadata`: 附件详情 (JSON)
